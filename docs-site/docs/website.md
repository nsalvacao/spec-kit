# Website Plan (GitHub Pages)

## Framework Decision

**Selected:** MkDocs Material  
**Why:** fast to scaffold, docs-first, clean navigation, minimal build complexity, and excellent GitHub Pages support.

## Site Structure (Outline)

- **Landing**: fork value proposition, Phase 0 overview, call-to-action  
- **Install**: CLI install options + template override  
- **Quickstart**: end-to-end example  
- **Phase 0**: ideate/select/structure/validate  
- **Compatibility**: upstream sync + fork guarantees  
- **Releases**: versioning + changelog  
- **Governance**: license, security, support, trademarks
